# GitHub Agentic Workflow

[claude-code-action@v1](https://github.com/anthropics/claude-code-action) 기반의 자동화된 개발 파이프라인입니다. 이슈에 `agent` 라벨을 붙이면 Plan -- Implement -- Review 3단계가 자동으로 실행됩니다.

## 파이프라인 개요

```
Issue + `agent` label
    → [Workflow 1: Issue Planner] → Plan comment + `plan-complete`
        → [Workflow 2: Agent Implement] → PR + `needs-review`
            → [Workflow 3: PR Reviewer] → Review loop (max 3x) → `review-complete`
```

## Workflow 상세

### 1. Issue Planner

`agent` 라벨이 추가되면 트리거됩니다.

**동작:**
- 이슈 내용을 분석하여 구조화된 구현 계획을 코멘트로 작성
- 파일 변경 목록, 테스트 계획, 영향 범위를 포함
- 정보가 부족하면 `needs-clarification` 라벨을 추가하고 질문 코멘트 작성
- 계획이 완성되면 `plan-complete` 라벨 추가

### 2. Agent Implement

`plan-complete` 라벨이 추가되면 트리거됩니다.

**동작:**
- Plan 코멘트를 읽고 코드 구현 시작
- 빌드 및 린트 명령 실행하여 코드 품질 확인
- 테스트 통과 확인
- PR 생성 후 `needs-review` 라벨 추가

### 3. PR Reviewer

`needs-review` 라벨이 추가되면 트리거됩니다.

**동작:**
- PR diff를 분석하여 리뷰 코멘트 작성
- HIGH 이슈가 발견되면 자동 수정 시도 (최대 3회 반복)
- 모든 이슈가 해결되면 `review-complete` 라벨 추가
- 최종 리뷰 요약 코멘트 작성

## 레포별 설정 차이

| | aerospike-py | ACKO | cluster-manager |
|---|---|---|---|
| **Language** | Python / Rust | Go | Python / TypeScript |
| **Build cmd** | `make build` | `go build ./...` | `ruff check` / `npm run type-check` |
| **Test cmd** | `make test-unit` | `go vet ./...` | -- |
| **Lint** | `ruff check` / `cargo clippy` | `golangci-lint run` | `ruff check` / `eslint` |

## 안전 장치 (Safety Guards)

파이프라인의 안정적인 운영을 위해 다음 안전 장치가 적용됩니다:

| Guard | 설정 | 설명 |
|-------|------|------|
| **최대 리뷰 반복** | 3회 | PR Reviewer가 수정을 시도하는 최대 횟수 |
| **코멘트 할당량** | 제한 적용 | API 비용 제어를 위한 코멘트 수 제한 |
| **타임아웃** | 워크플로우별 제한 | GitHub Actions 타임아웃으로 무한 실행 방지 |
| **라벨 기반 제어** | 수동 라벨 제거로 중단 | `agent` 라벨을 제거하면 파이프라인 중단 |

:::warning 주의사항
- `agent` 라벨은 충분한 컨텍스트가 있는 이슈에만 추가하세요
- 복잡한 크로스-레포 변경은 Epic으로 분리한 후 개별 레포에서 `agent` 라벨을 사용하세요
- 리뷰 루프가 3회를 초과하면 수동 개입이 필요합니다
:::

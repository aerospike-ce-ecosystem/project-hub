# GitHub Agentic Workflow

[claude-code-action@v1](https://github.com/anthropics/claude-code-action) 기반의 자동화된 개발 파이프라인입니다. 이슈에 `agent` 라벨을 붙이면 Plan -- Implement -- Review 3단계가 자동으로 실행됩니다.

## 파이프라인 개요

```
Issue + `agent` label
    → [Workflow 1: Issue Planner] → Plan comment + `plan-complete`
        → [Workflow 2: Agent Implement] → PR + `needs-review`
            → [Workflow 3: PR Reviewer] → Review loop (max 3x) → `review-complete`
```

## Workflow 상세

### 1. Issue Planner

`agent` 라벨이 추가되면 트리거됩니다.

**동작:**
- 이슈 내용을 분석하여 구조화된 구현 계획을 코멘트로 작성
- 파일 변경 목록, 테스트 계획, 영향 범위를 포함
- 정보가 부족하면 `needs-clarification` 라벨을 추가하고 질문 코멘트 작성
- 계획이 완성되면 `plan-complete` 라벨 추가

### 2. Agent Implement

`plan-complete` 라벨이 추가되면 트리거됩니다.

**동작:**
- Plan 코멘트를 읽고 코드 구현 시작
- 빌드 및 린트 명령 실행하여 코드 품질 확인
- 테스트 통과 확인
- PR 생성 후 `needs-review` 라벨 추가

### 3. PR Reviewer

`needs-review` 라벨이 추가되면 트리거됩니다.

**동작:**
- PR diff를 분석하여 리뷰 코멘트 작성
- HIGH 이슈가 발견되면 자동 수정 시도 (최대 3회 반복)
- 모든 이슈가 해결되면 `review-complete` 라벨 추가
- 최종 리뷰 요약 코멘트 작성

## 안전 장치 (Safety Guards)

파이프라인의 안정적인 운영을 위해 다음 안전 장치가 적용됩니다:

| Guard | 설정 | 설명 |
|-------|------|------|
| **최대 리뷰 반복** | 3회 | PR Reviewer가 수정을 시도하는 최대 횟수 |
| **코멘트 할당량** | 제한 적용 | API 비용 제어를 위한 코멘트 수 제한 |
| **타임아웃** | 워크플로우별 제한 | GitHub Actions 타임아웃으로 무한 실행 방지 |
| **라벨 기반 제어** | 수동 라벨 제거로 중단 | `agent` 라벨을 제거하면 파이프라인 중단 |

:::warning 주의사항
- `agent` 라벨은 충분한 컨텍스트가 있는 이슈에만 추가하세요
- 복잡한 크로스-레포 변경은 project-hub에서 Cross-Repo Issue 또는 Epic으로 관리하세요
- 리뷰 루프가 3회를 초과하면 수동 개입이 필요합니다
:::

## Hub Orchestration Layer

project-hub은 크로스-레포 이슈를 조율하는 **허브 오케스트레이터** 역할을 합니다. Issue template에 따라 자동으로 분석/계획을 수행하고, 승인 후 각 repo에 issue를 자동 생성합니다.

### ADR Proposal 흐름

```
ADR Proposal Issue 생성 (adr + discussion 라벨 자동 부착)
    → hub-issue-planner.yml (adr-review job) 트리거
        → AI가 기존 ADR/프로젝트 목표와 대조하여 검토
        → 3가지 판단 중 하나를 코멘트로 작성:
            ❌ REJECT (기각): adr-rejected 라벨
            ⏸️ DEFER (보류): adr-deferred 라벨
            ✅ POSITIVE (긍정적 검토): adr-positive 라벨 + repo별 영향 분석
    → [사람이 최종 판단]
    → start-implement 라벨 추가 (adr-positive 필수)
        → hub-issue-dispatcher.yml 트리거
            → 각 repo에 issue 자동 생성 + agent 라벨
            → 각 repo의 기존 파이프라인 동작
```

### Cross-Repo Issue / Epic 흐름

```
Cross-Repo Issue 또는 Epic 생성 (cross-repo / epic 라벨 자동 부착)
    → hub-issue-planner.yml 트리거
        → AI가 각 repo 코드베이스를 탐색하여 구현 계획 수립
        → repo별 구현 계획을 코멘트로 작성
        → plan-complete 라벨 추가
    → [사람이 계획 검토]
    → start-implement 라벨 추가
        → hub-issue-dispatcher.yml 트리거
            → 각 repo에 issue 자동 생성 + agent 라벨
            → 각 repo의 기존 파이프라인 동작
```

### Hub ↔ Core Repo 연결

Hub dispatcher가 각 repo에 issue를 생성할 때 `agent` 라벨을 부착하므로, 해당 repo의 기존 **Issue Planner → Agent Implement → PR Reviewer** 파이프라인이 자동으로 시작됩니다.

```
project-hub                          각 Core Repo
┌──────────────────┐                ┌──────────────────────┐
│ Issue 생성        │                │                      │
│   ↓               │                │ issue-planner.yml    │
│ Hub Planner       │                │   ↓                  │
│   ↓               │                │ agent-implement.yml  │
│ [사람 검토]       │                │   ↓                  │
│   ↓               │   agent 라벨   │ pr-reviewer.yml      │
│ Hub Dispatcher  ──┼───────────────→│   ↓                  │
│   ↓               │                │ PR merge             │
│ dispatched 라벨   │                │                      │
└──────────────────┘                └──────────────────────┘
```

# Interactive Architecture Diagrams

Aerospike CE Ecosystem의 아키텍처를 인터랙티브 다이어그램으로 탐색할 수 있습니다.
각 다이어그램은 드래그, 줌, 패닝을 지원하며 노드 위에 마우스를 올리면 하이라이트됩니다.

:::tip 조작 방법
- **드래그**: 노드를 클릭하고 끌어서 위치 조정
- **줌**: 마우스 휠 또는 컨트롤 버튼 사용
- **미니맵**: 우측 하단의 미니맵으로 전체 구조 확인
- **노드 클릭**: GitHub 링크가 있는 노드는 클릭하면 해당 레포로 이동
- **확대**: 우측 상단 확대 버튼으로 전체 화면 보기
:::

## 2. aerospike-py Module Structure

aerospike-py의 내부 모듈 구조입니다. Python API 레이어에서 PyO3 브릿지를 거쳐 Rust 코어까지의
계층 구조를 보여줍니다.

- **Python Layer**: Client API, CDT operations, Expression filters, NumPy batch
- **PyO3 Bridge**: Python-Rust 바인딩 (pyo3 0.28.2)
- **Rust Core**: 비동기 클라이언트, wire protocol, backpressure
- **External**: aerospike-client-rust v2, Tokio runtime

## 3. ACKO Module Structure

Aerospike CE Kubernetes Operator(ACKO)의 내부 모듈 구조입니다.

- **CRD Layer**: AerospikeCluster custom resource definition
- **Controller Layer**: 22개의 controller 파일 (reconciler, aero_client, events 등)
- **Internal Packages**: configdiff, configgen, metrics 등 11개 패키지
- **Helm Charts**: Operator chart, CRD subchart
- **Managed Resources**: StatefulSet, Service, PVC, ConfigMap, PDB, NetworkPolicy
